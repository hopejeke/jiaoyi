# Stripe 测试卡号指南

## ⚠️ 重要提示

**在 Stripe 测试环境（Sandbox）中，绝对不要使用真实信用卡号！**

Stripe 提供了专门的测试卡号，用于模拟各种支付场景。

## 📝 测试卡号的通用信息

所有测试卡号使用以下信息：

- **过期日期（Expiry Date）**：任意未来的日期，例如 `12/25` 或 `01/26`
- **CVC/CVV**：任意 3 位数字，例如 `123` 或 `456`
- **ZIP 码（如需要）**：任意 5 位数字，例如 `12345`
- **持卡人姓名（如需要）**：任意姓名，例如 `Test User`

结果：支付成功 ✅

## 🔍 如何验证测试环境

1. **检查 Stripe Dashboard**：
   - 确保右上角显示"沙盒"（Sandbox）或"测试"（Test）
   - URL 中包含 `/test/` 路径

2. **检查 API Key**：
   - 测试环境的 API Key 以 `pk_test_` 和 `sk_test_` 开头
   - 生产环境的 API Key 以 `pk_live_` 和 `sk_live_` 开头

3. **查看测试交易**：
   - 在 Stripe Dashboard 的"交易"（Payments）页面
   - 所有测试交易都会显示，不会产生真实扣款

## ⚠️ 安全提示

1. **永远不要在测试环境使用真实卡号**
2. **确保生产环境使用 `pk_live_` 和 `sk_live_` 开头的 API Key**
3. **测试环境不会产生真实扣款**
4. **测试卡号只在测试环境有效，在生产环境会被拒绝**
